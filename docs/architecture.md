# 校园论坛项目架构文档

## 一、项目概述

本项目是一个面向高校学生和教职工的综合性校园论坛平台，旨在提供信息交流、资源分享、社交互动等功能。

### 1.1 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端框架 | HTML5 + CSS3 + JavaScript (ES6+) | 原生前端技术，无需构建工具 |
| 样式规范 | Google Material Design 3 (Material You) | 遵循 Material Design 设计规范 |
| 图标库 | Google Material Symbols | 统一图标风格 |
| 数据库 | Supabase PostgreSQL | 云原生 PostgreSQL 数据库 |
| 认证 | Supabase Auth | 用户认证与授权 |
| 图片存储 | Supabase Storage | 图片和文件存储 |
| 部署 | GitHub Pages | 静态页面托管 |

### 1.2 设计原则

- **模块化**: 功能按模块划分，高内聚低耦合
- **可维护**: 代码结构清晰，易于理解和维护
- **可扩展**: 架构设计支持未来功能扩展
- **安全优先**: 严格的权限控制和数据安全措施
- **响应式**: 支持移动端和桌面端访问

## 四、数据流向

### 4.1 认证流程

```
用户登录请求
    ↓
auth.js (认证服务)
    ↓
Supabase Auth API
    ↓
返回用户信息 + Token
    ↓
storage.js (存储到本地)
    ↓
更新页面状态
```

### 4.2 数据请求流程

```
页面发起请求
    ↓
services/xxx.js (业务服务)
    ↓
services/api.js (通用API)
    ↓
Supabase Client
    ↓
Supabase Database
    ↓
返回数据
    ↓
页面渲染
```

### 4.3 文件上传流程

```
选择文件
    ↓
压缩图片 (helpers.js)
    ↓
services/storage.js
    ↓
Supabase Storage API
    ↓
返回文件URL
    ↓
保存到数据库
```

## 五、安全架构

### 5.1 权限模型

| 角色 | 权限级别 | 说明 |
|------|----------|------|
| 游客 | 0 | 浏览帖子、搜索 |
| 普通用户 | 1 | 发帖、评论、私信 |
| VIP用户 | 2 | 高级功能（预留） |
| 版主 | 3 | 管理板块、审核内容 |
| 管理员 | 4 | 管理用户、系统设置 |
| 超级管理员 | 5 | 全部权限 |

### 5.2 安全措施

- **RLS (Row Level Security)**: 数据库行级权限控制
- **SQL注入防护**: 使用参数化查询
- **XSS防护**: 内容过滤和转义
- **CSRF防护**: 会话令牌验证
- **密码加密**: Supabase Auth 自动处理
- **图片安全**: 上传前格式和大小验证
- **登录限流**: 防止暴力破解

## 七、架构设计原则

### 7.1 SOLID原则

- **单一职责**: 每个模块只负责一个功能
- **开闭原则**: 对扩展开放，对修改关闭
- **里氏替换**: 子类可以替换父类
- **接口隔离**: 客户端只依赖需要的接口
- **依赖倒置**: 依赖抽象而非具体实现

### 7.2 设计模式

- **模块模式**: 使用 ES Module 组织代码
- **服务模式**: 业务逻辑封装为服务
- **工厂模式**: 统一创建对象
- **观察者模式**: 状态变化通知
