## 六、已知问题

### 6.1 数据库迁移未执行

**描述**: `post_images` 表尚未在 Supabase 中创建，导致图文帖子的图片关联存储功能无法正常工作。

**影响**: 纯文字帖子不受影响，图文帖子的图片关联合写入会失败。

**解决方案**: 在 Supabase SQL Editor 中执行迁移文件。

### 6.2 Supabase 配置未更新

**描述**: `config/supabase.js` 中的 SUPABASE_URL 和 SUPABASE_ANON_KEY 仍为示例值。

**影响**: 所有与 Supabase 的交互将无法正常工作。

**解决方案**: 替换为真实的项目配置。

### 6.3 Storage Bucket 名称一致性

**描述**: storageService 现在使用配置中的 `postImageStoragePath`（默认值为 `post-images`）。

**影响**: 如果 Supabase 中创建的 bucket 名称不一致，图片上传将失败。

**解决方案**: 在 Supabase 中创建名为 `post-images` 的 Storage bucket，或修改配置中的 `postImageStoragePath` 值。

### 6.4 图片上传失败处理策略

**描述**: 当前策略是帖子创建成功但图片上传失败时，保留帖子并提示用户。

**影响**: 帖子可能缺少图片，用户体验受影响。

**后续优化**: 
- 考虑增加失败回滚机制
- 或增加图片重试上传功能

### 6.5 缺少事务支持

**描述**: 当前发帖流程（createPost → uploadPostImages → savePostImages）不是原子操作。

**影响**: 在极端情况下可能出现数据不一致。

**后续优化**: 考虑使用 Supabase Edge Functions 实现服务端事务。
