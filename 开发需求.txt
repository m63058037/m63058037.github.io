# 校园论坛项目开发需求文档（正式版）

这是一个面向在校大学生的校园论坛平台，旨在为学生提供一个交流、分享和学习的社区。

## 一、项目简介

开发一个适用于高校校园社区的论坛平台。

核心功能：
- 用户注册和登录
- 帖子发布、浏览和管理
- 评论互动
- 用户之间的关注和私信
- 内容审核和权限管理

---

# 二、技术栈

前端：
- HTML5
- CSS3
- JavaScript (ES6+)

数据库：
- Supabase PostgreSQL

认证：
- Supabase Auth

存储：
- Supabase Storage

部署：
- GitHub Pages

版本管理：
- Git + GitHub

# 十三、数据库

采用标准关系型数据库。

主要表：
- users
- profiles
- posts
- comments
- likes
- favorites
- categories
- notifications
- reports
- admin_logs

设计：
- 主键
- 外键
- 索引
- RLS规则

不得与用户权限冲突。

---

# 十四、数据库迁移

功能：
- 数据库迁移不会影响用户数据。

核心：
- 数据库设计归属于前端
- 数据迁移
- 结构迁移
- SQL迁移
- JSON迁移
- 数据库备份
- 迁移前数据备份
- 数据库重建（Migration）
- 版本数据库设计不得修改旧数据库

原则：
- 所有迁移方案都必须备份数据库。

- admin.js
- storage.js
- search.js

# 十六、开发规范

功能：
- 遵循单一职责原则
- 避免 Bug 后修复
- 保证数据完整性
- 统一代码规范
- 测试驱动开发
- 不进行不必要的功能开发
- 所有功能统一测试
- 不随意修改数据

---

# 十七、开发流程

原则一个阶段完成一个模块。

按照如下的顺序进行开发：

第一阶段：
- 设计项目架构。

第二阶段：
- 设计数据库。

第三阶段：
- 设计UI。

第四阶段：
- 开发认证功能。

第五阶段：
- 开发论坛功能。

第六阶段：
- 开发管理功能。

第七阶段：
- 测试。

最后一个阶段优化，然后进行上线前修复。

同时开发文档，完成修改后提交验收。
